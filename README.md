# 📌 1. 프로젝트 개요
| 항목 | 내용 |
| :--- | :--- |
| **개발 기간** | 2025.06.18 ~ 2025.07.18 (4주) |
| **팀 구성** | 6인 팀 프로젝트 (프론트엔드 3, 백엔드 3) |
| **내 역할** | 백엔드: 중계 일정 / 검색 API 개발, **k6 부하 테스트 & Redis 캐싱 전략 수립 및 구현** |
| **배포 서버** | [http://3.35.146.155:3000](http://3.35.146.155:3000) |

## 1.1 기획 의도
대중적인 지도 서비스에는 **스포츠 중계 필터**가 없어 스포츠 팬들이 함께 관람할 장소를 찾기 어렵습니다. 또한 식당 또한 경기를 중계해도 이를 알릴 방법이 부족합니다. **PLAYCE**는 이러한 정보 불일치를 해소하기 위해 기획되었습니다.

## 1.2 프로젝트 소개
> **PLAYS + PLACE = PLAYCE**

PLAYCE는 사용자 주변의 **스포츠 중계 식당을 쉽게 탐색**할 수 있도록 돕는 지도 서비스입니다. 식당 위치, 경기 종목, 팀 등을 바탕으로 최적의 장소를 검색할 수 있으며, **기존 지도 서비스에는 없던 스포츠 경기 중계 일정을 확인**할 수 있습니다.

## 1.3 활용 방안 및 기대 효과
* **스포츠 팬의 장소 검색 도구**: 경기 일정과 위치 기반으로 편리하게 장소 탐색 가능.
* **식당의 마케팅으로 활용**: 중계 일정을 등록하여 새로운 고객 유입 및 매출 극대화 가능.
* **스포츠 문화 활성화**: 함께 응원하며 경기를 시청하는 문화를 활성화하고 커뮤니티 플랫폼으로 확장 가능.

# 🚀 2. 주요 기능
| 기능 분류 | 설명 |
| :---: | :--- |
| **지도 및 검색** | - **카카오맵 API** 기반 지도 서비스 제공. <br> - 마커 클릭 시 식당 기본 정보 확인 가능. <br> - **통합 검색**: 식당 이름, 지역, 경기(종목, 리그)별 필터링 및 거리순/날짜순 정렬 제공. |
| **식당 상세 정보** | - 지도 마커 또는 검색 결과 클릭 시 진입.<br> - 기본 정보, 메뉴, 사진, **중계 일정** 등 상세 정보 확인. |
| **마이페이지** | - 로그인 시 접근 가능. 즐겨찾기 목록 조회/삭제. 내 정보 조회. |
| **식당/일정 관리** (Admin) | - **식당 등록**: 사업자등록번호, 주소 등 유효성 검사 진행. <br> - **중계 일정 관리**: 탭 리스트 및 캘린더를 통한 일정 등록, 수정, 삭제. |

# 🛠️ 3. 기술 스택 및 아키텍처
## 3.1 기술 스택
| 구분 | 내용 |
| :--- | :--- |
| **프론트엔드** | **TypeScript**, React, **Zustand**, **Tailwind CSS** |
| **백엔드** | **TypeScript**, Node.js, Express, **MySQL**, **TypeORM** |
| **주요 서비스** | **카카오맵 API**, **AWS S3** (이미지), **Redis** (캐싱), **k6** (부하 테스트), **AWS CloudWatch** (모니터링), **AWS EC2** (배포) |

## 3.2 아키텍처
성능 최적화를 위해 **Redis**를 캐싱 계층으로 도입하여 DB 부하를 분산하고, AWS EC2 인스턴스에 배포했습니다.
